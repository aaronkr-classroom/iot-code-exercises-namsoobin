# chp1 연습문제

1.Q1
  - (a) 디바이스의 센서와 액추에이터의 역할을 비교하여 설명하세요.
    - 센서는 다양한 조건을 감지하여 수치나 신호로 출력하는 장치입니다. 예시로는 차량의 전, 후 측면에 사용되어 충돌을 감지, 후진 시 장애물이 가까워지면 경고음을 내는 주차 시스템에 이용되는 초음파 센서가 있습니다. 다만 측정범위가 짧고 환경의 영향을 많이 받는다는 단점이 있습니다.
    - 동력의 근본이 되는 것을 총칭해 액추에이터라고 합니다. 예시로는  직류전류를 전원으로 하는 DC모터(직류모터)는 주로 정밀기계나 건전지로 움직이는 장난감등에 활용됩니다.
    - 요약하자면, 센서는 디바이스 주위의 상태를 감지하고 액추에이터는 디바이스가 움직이게 만드는 장치라는 점에서 차이가 있습니다.
    - (https://kr.meviy.misumi-ec.com/info/ko/archives/32278/), (https://www.hlworld.com/403)
  - (b) 디바이스와 대시보드에서 동일한 프로그래밍 언어를 사용하지 않는 이유를 설명하세요.
     - 각 층이 요구하는 성능·개발 속도·생태계가 크게 다르기 때문에 둘은 동일한 프로그래밍 언어를 사용하지 않습니다. 속도가 중요한 디바이스는 하드웨어에 가깝기 때문에 저급 언어를 쓰고, 대시보드는 개발자의 편의성을 위해 고급 언어를 씁니다.
     - (https://syp.vn/kr/article/best-languages%E2%80%8B-for-AI-enabled-web-apps), (https://wikidocs.net/blog/@jaehong/8474/)
  - (c) 브로커와 같은 정보 전달 전담 서버의 이점을 설명하세요.
     - 브로커를 사용하면, 순간적으로 트래픽이 폭증해도 큐를 이용해 완충 역할을 합니다. 또한 수신 서비스가 잠시 죽거나 느려져도, 브로커는 메시지를 보관했다가 다시 살아났을 때 전달할 수 있습니다. 이로써 데이터 유실을 줄이고 메시지를 안정적으로 전달할 수 있게 합니다.
     - (https://dgjinsu.tistory.com/99), (https://binux.tistory.com/74)

  - (c) Mosquitto와 Node-RED에 대해 설명, 실행에 적합한 컴퓨팅 환경 제시
    - Mosquitto는 가벼운 MQTT 브로커이고, Node‑RED는 흐름(플로우) 기반의 시각 프로그래밍 도구입니다.
    - Mosquitto는 CPU는 라즈베리 4, RAM은 1GB, 네트워크는 유선 LAN이 권장됩니다.
    - Node-RED는 CPU는 2코어, RAM은 2GB, 저장공간은 8GB를 권장합니다.
    - (https://12340zszs.tistory.com/70)
